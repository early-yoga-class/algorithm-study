## ✏️ 규칙

- 토요일 스터디 시간에 문제 2개씩(제한시간 90분) 풀고 풀이 공유
- 토요일 오전 9시까지 `해볼까?` 주제 구현해서 pr 올리기, 코드리뷰
- 다음 주 문제 선정 / `해볼까?` 주제 선정

<br>

## 🍴 저장소 규칙

### 브랜치 네이밍

```
week{주차}-문제이름 or 해볼까 주제 이름

ex) week02-모음사전
ex) week02-힙
```

<br>

### pr 규칙

```
pr 제목 : [이름] - 몇주차 제출

label - [해볼까? or 알고리즘]

ex) [창현] - 1주차 제출
오른쪽 labels 탭에서 관련 라벨 선택
내용은 관련 주제나 문제 접근 방식 설명
```

<br>

### commit message 규칙

```
태그 : 내용

ex) 해볼까? 힙 구현 코드 업로드시
Add: 힙 해볼까? 박창현 (Add: 주제 + 해볼까? + 이름)
```

| 태그    | 설명                                                  |
| ------- | :---------------------------------------------------- |
| Add     | 새로운 문제풀이 추가                                  |
| Style   | 코드 포맷 변경, 세미 콜론 누락, 코드 수정이 없는 경우 |
| Comment | 필요한 주석 추가 및 변경                              |
| Docs    | 문서 수정(README.md 포함)                             |
| Rename  | 파일 혹은 폴더명을 수정하거나 옮기는 작업만인 경우    |
| Remove  | 파일을 삭제하는 작업만 수행한 경우                    |

<br>

1. 새로운 branch 생성

3. push 완료 후 GitHub branch 페이지에서 Pull Request(PR) 버튼 클릭
